# Phase 09.12 Plan 04: Invite Expiry Warning UI Summary

## Objective
Add UI warning for invites expiring within 1 day. Previously invites had a 7-day hardcoded expiration with no user notification.

## One-liner
Invite expiry warning banner with countdown timer shown when pending invite is within 24 hours of expiration.

## Changes Made

### 1. SecureTokenStore - PendingCompanyInvite helpers
Added to `lib/features/company_auth/data/services/secure_token_store.dart`:
- `expiryDuration` (7 days) - constant for invite TTL
- `warningThreshold` (1 day) - when to show warning
- `isExpired` - checks if invite has expired
- `isExpiringSoon` - checks if within warning window
- `expiresAt` - computed expiry DateTime
- `remainingTime` - human-readable countdown ("6 horas", "1 dia")

- Dismissible warning banner
- Shows warning icon and message
- Displays remaining time from invite
- Uses error tone colors (errorContainer background)
- Close button for dismissal

### 3. CompanyShell - Invite expiry check
Added to `lib/app/navigation/company_shell.dart`:
- `_pendingInvite` state field
- `_showInviteWarning` visibility flag
- `_inviteWarningDismissed` session dismissal flag
- `_checkPendingInviteExpiry()` method called on init
- Warning banner displayed above profile area when conditions met
- Silent cleanup of expired invites

## Verification

User sees warning when:
- User has a pending invite
- Invite expires within 24 hours
- Warning shows remaining time ("expira em 6 horas", "expira em 1 dia")
- Warning can be dismissed for the session
- Warning reappears on app restart if still valid

## Deviations from Plan

### Architecture Adaptation
The plan specified adding pending invite checking to `AdminEmpresaViewModel`, but this view model is specifically for team invite management (admin area), not for checking the user's own pending invite. The implementation was moved to `CompanyShell` which:
- Is the parent navigation container
- Has direct access to `SecureTokenStore` via `AppScope`
- Wraps `CompanyHomeScreen` in the profile tab
- Allows the warning to appear above any navigation area

This is a better architectural choice as the warning is tied to user authentication state, not team administration.

### CompanyHomeScreen Direct Integration
Instead of modifying `CompanyHomeScreen` directly, the warning is displayed by `CompanyShell` wrapping the profile area. This keeps `CompanyHomeScreen` unchanged and centralizes the warning logic.

## Threat Flags
None - this change only adds client-side warning UI and does not affect security boundaries.
