## Test Coverage

### Backup/Restore Tests (backup_restore_test.dart)

**Backup Export:**
- Creates valid backup file with RAT data
- Includes checksums for integrity validation
- Handles empty backup (no RATs)
- Includes assinaturas when present

**Backup Restore:**
- Restores RATs to repository correctly
- Handles legacy JSON format
- Preserves RAT status and sync status
- Handles multiple RATs with different owners

**Integrity Validation:**
- Validates unmodified backup as valid
- Detects tampered content
- Rejects corrupted ZIP format
- Rejects backup with missing manifest

**Conflict Resolution:**
- Handles same ID with different content (newer timestamp wins)
- Preserves data from both sources on restore

**Edge Cases:**
- Handles RAT with null optional fields
- Handles large number of RATs (100 items)
- Handles special characters in client name

### Sync Flow Tests (sync_flow_test.dart)

**Offline Create to Online Sync:**
- Offline RAT saved locally with pendingSync status
- EnqueueSync adds RAT to sync queue
- ProcessSyncQueue sends pending RAT to remote
- Successful sync updates RAT local status to synced

**Conflict Resolution:**
- Local vs remote conflict with newer timestamp resolution
- Deduplicates sync queue for same RAT

**Retry Failed Sync:**
- Failed item retried when retryFailed=true
- Failed item not retried when retryFailed=false
- Failed sync increments attempt counter
- Max attempts exceeded marks item as permanently failed

**Sync Queue Processing:**
- Processes multiple items in sequence
- Failed item does not block other items
- Deletes are processed correctly

**Queue Statistics:**
- countPending returns correct count
- Items filter by empresa and usuario correctly

### Auth Flow Tests (auth_flow_test.dart)

**Sign In:**
- Successful sign in returns remote session
- Successful sign in saves tokens to secure store
- Successful sign in saves session to repository
- Sign in with invalid credentials throws RemoteAuthException
- Sign in does not save tokens on failure
- Sign in clears previous session

**Sign Up with Invite:**
- Stores pending invite when confirmation required
- Clears pending invite on success

**Session Restore:**
- Returns session when tokens exist
- Returns null when no tokens
- Returns null when no session saved
- Updates tokens on refresh

**Session Refresh:**
- Returns session when tokens exist
- Updates tokens
- Returns null when no session
- Returns null when tokens cleared
- Returns null when refresh fails

**Sign Out:**
- Clears all tokens and session
- Completes even if remote sign out fails
- No active session does nothing

**Error Handling:**
- Sign in with empty email shows friendly error
- Maps invalid_credentials to friendly message
- Does not leak internal error details
