# Phase 9.5: Code Review Fixes — Specification

**Created:** 2026-06-15

## Goal

Fix all code review findings (1 critical, 3 warnings, 3 informational) identified in REVIEW.md, ensuring zero critical issues and warnings remain before release.

## Background

Code review via `/gsd-code-review` identified 7 issues across 18 modified/new files. One critical issue (CR-01) can cause data inconsistency in company mode RATs. Three warnings (WR-01 to WR-03) affect code quality and performance. Three informational items (IN-01 to IN-03) are improvements with lower priority.

## Requirements

1. **CR-01: Error handling in reopenForCorrection**: Wrap `save()` call in try-catch to ensure state rollback executes even if sync fails.
   - Current: State is modified before `save()`, but rollback only runs if `save()` returns false — unhandled exceptions bypass rollback
   - Target: All exceptions from `save()` trigger rollback and set error message
   - Acceptance: If `save()` throws (e.g., sync failure), state is rolled back and `_errorMessage` is set; no inconsistent state remains

2. **WR-01: UpdateTecnicoLocal injection**: Move `UpdateTecnicoLocal` instantiation from `_saveEditing()` to constructor or class field.
   - Current: New `UpdateTecnicoLocal` instance created on every save attempt
   - Target: `UpdateTecnicoLocal` is injected via constructor or created once in `initState()`
   - Acceptance: `UpdateTecnicoLocal` is not instantiated inside `_saveEditing()` method

3. **WR-02: shouldRepaint optimization**: Implement actual comparison in `shouldRepaint()` instead of always returning `true`.
   - Current: `_SignaturePainter.shouldRepaint` always returns `true`, causing unnecessary repaints
   - Target: `shouldRepaint` compares `strokes.length` and each stroke's length
   - Acceptance: Signature capture UI only repaints when strokes actually change (verified via debug or frame count)

4. **WR-03: Magic number as constant**: Define `maxSignatureBytes` as a class/static constant instead of inline literal.
   - Current: `const maxSignatureBytes = 1 * 1024 * 1024;` is defined inline in method
   - Target: `maxSignatureBytes` is a class constant in `RatFormViewModel`
   - Acceptance: Constant is accessible as `RatFormViewModel.maxSignatureBytes` and is used in validation

5. **IN-01: Comment in Portuguese**: Replace Chinese characters in test file comment with Portuguese.
   - Current: Comment uses "bytes伪造" (Chinese) mixed with Portuguese
   - Target: Comment uses Portuguese only ("bytes simulados")
   - Acceptance: Test file compiles without the Chinese character in comment

6. **IN-02: UUID for signature ID** (optional/deferrable): Consider using UUID instead of `microsecondsSinceEpoch` for signature IDs.
   - Current: Signature IDs use `microsecondsSinceEpoch` which has theoretical collision risk
   - Target: Signature IDs use UUID (or explicit decision to keep microseconds)
   - Acceptance: Either UUID is implemented, OR decision to keep microseconds is documented with rationale

7. **IN-03: Fire-and-forget pattern documented**: Add explicit documentation or use `unawaited()` pattern for the non-awaited delete call.
   - Current: `_enqueueAssinaturaSync?.delete()` is not awaited; comment explains intent but pattern is implicit
   - Target: Either use `unawaited()` wrapper, or add `// ignore: unused_result` with explanatory comment
   - Acceptance: Code is more explicit about intentional fire-and-forget behavior
